# GG Tesla App Platform

Монорепозиторий для event-driven платформы с web-клиентом, core API, микросервисами реального времени и инфраструктурой деплоя.

> Важно: `Tesla` в названии репозитория — это внутренний бренд/кодовое имя проекта, а не обязательная привязка к автомобилям Tesla.

## Зачем это нужно

Проект собирает в одном месте базовые строительные блоки продуктовой платформы:
- **Core API (HELIO)** для HTTP-интеграций и оркестрации.
- **Микросервисы на Rust** для real-time и тяжелых фоновых задач.
- **Web/бот клиенты** как точки входа для пользователей.
- **Infra-as-code** (Helm, ArgoCD, Vault, OPA) для production-развертывания.

## Для кого это

- **Backend/Platform инженеры** — развивать API, события и интеграции.
- **Frontend инженеры** — развивать web-приложение в `apps/web`.
- **DevOps/SRE** — управлять деплоем и политиками доступа.
- **AI/Automation инженеры** — добавлять ассистентов и автоматизации.

## Текущая структура репозитория

- `core/helio` — Node.js/Fastify core API.
- `microservices/*` — Rust сервисы (`sfu`, `crdt-sync`, `reporting`).
- `apps/web` — React/Vite клиент.
- `apps/telegram-bot` — Telegram интерфейс.
- `tesla_app` — отдельный Python CLI-модуль (демо-интеграция с Tesla API).
- `infra`, `helm`, `argocd` — инфраструктура и манифесты деплоя.

## Почему есть `tesla_app`

`tesla_app` — это изолированный Python-модуль/демо для AI-команд через CLI. Он не определяет весь смысл монорепозитория и может использоваться как пример AI-интеграции с внешним API.
